## Problem Statement

QA 工程师写测试用例是一个高认知负荷的手工流程：拿到 PRD → 拆解测试点 → 写出覆盖正向/边界/异常场景的结构化用例。一个中等功能模块，50-100 条用例可能花一整天。质量高度依赖个人经验——初级 QA 漏边界值，中级 QA 可能漏异常路径，高级 QA 太贵。

目前没有一款产品专门解决"需求 → 结构化测试用例"这个环节。TestRail/Qase/Zephyr 管用例但不帮你写，ChatGPT 能写但输出不结构化、不专业、需要大量手动整理。

AI 的价值不在于替代 QA，而是把"从 0 到 60 分"压缩到几秒钟，QA 做"60 到 100 分"的审核补充。

## What Makes This Cool

- 需求粘贴进去 + 一张截图，30 秒出 50 条结构化用例——有编号、前置条件、步骤、预期结果、优先级、用例类型
- AI 主动标记"规则模糊的地方"，提醒 QA 需要跟产品确认的点
- 导出就是标准 Excel/文本格式，直接能用，不用改格式
- Prompt 由真正做过测试的人调优，输出质量远超通用 AI 聊天

## Target Users

- **第一梯队（冷启动）:** 国内 1-5 人测试团队、独立 QA 顾问、测试群里活跃的从业者
- **第二梯队（扩展）:** 测试经理/QA Lead（有预算权）、中小公司技术负责人
- **出海后:** 英语市场的 QA 个人和小团队，Product Hunt 获客

## Constraints

- 一个人开发，Claude Code 辅助编码
- MVP 不做登录/注册/团队协作
- 先中文 UI/Prompt，代码中硬编码中文文案（后续国际化时统一替换为 i18n key——改造成本低）
- AI 模型不绑定厂商——可插拔架构，改环境变量切换，月成本 ¥6-300 取决于所选模型（详见 AI 模型成本对比）
- 部署在 Vercel Pro（$20/月，需要 30 秒函数超时——Hobby 免费层只有 10 秒，不够 Claude API 调用）
- Vercel Pro 是唯一非零成本项，整体月运营成本约 ¥200-300

## Premises

1. QA 工程师"从需求写用例"是真实痛点，且无专用工具解决 → 已验证（用户自身 10 年 QA 经验确认）
2. Claude/GPT-4o 视觉+文本能力足以生成质量可靠的测试初稿 → 需 MVP 实测验证，关键风险点
3. 用户作为 QA 背景，能判断 AI 输出质量并调优 Prompt → 确定性高
4. MVP 先做 Web，不做登录/付费/协作 → 策略确认
5. 先中文版面向国内 QA 社区，验证后国际化出海 → 确认

## Approaches Considered

### Approach A: 极简单页（1-2 天）
单 HTML + 一个 API 端点。无保存、无历史、无导出。最快验证"有没有人用"。
→ 未被选择：对日常使用来说砍得太狠，不能保存和导出用户就留不住。

### Approach B: 功能版 Web（5-8 天）★ 选中
Next.js 全栈。输入页、结果展示编辑、导出 Excel、localStorage 存历史。匿名用量统计。代码结构预留用户认证插入点（API Route 不假设当前用户身份，后续加 NextAuth 不改路由层）。
→ 被选中：功能完整到可日常使用，技术复杂度可控，Claude Code 能搞定大部分代码。

### Approach C: Chrome 插件优先（2-3 天）
Chrome 扩展，右键选中文字 → 侧边栏出用例。工作流最紧密。
→ 未被选择但有价值：编辑受限，审核流程不友好。后期作为 Web 的快捷入口补充。

## Recommended Approach

**方案 B（功能版 Web）+ 后期补充 Chrome 插件作为快捷入口**

### MVP 功能清单

**输入层：**
- 文本输入框（粘贴 PRD/需求描述，支持 Markdown，最大 10,000 字符）
- 图片粘贴（Ctrl+V 粘贴截图到输入区，自动识别并展示缩略图，最多 3 张，单张 ≤ 5MB。第 4 张粘贴时弹出提示"已达上限"）
- 输入验证：提交前检查非空，超过字数限制截断并提示

**AI 处理层（可插拔模型架构）：**
- 后端 AI 调用层做 Provider 抽象——不绑定任何特定模型厂商
- 所有主流模型厂商都兼容 OpenAI SDK 格式（`baseURL` + `apiKey` + `model`），切换只需改环境变量
- Provider 接口统一：`generateTestCases(text: string, images: base64[]) → TestCaseResult`
- 环境变量控制当前使用的模型：
  ```
  AI_PROVIDER=deepseek|qwen|gemini|claude|openai   # 厂商标识（用于 Prompt 微调和日志）
  AI_BASE_URL=https://api.deepseek.com/v1          # API 地址
  AI_API_KEY=sk-xxx                                  # API Key
  AI_MODEL=deepseek-v4                               # 模型名
  ```
- System Prompt 存放为独立文件（`prompts/test-case-generator.md`），不硬编码在代码中，方便针对不同模型微调不同版本
- JSON Schema 对所有模型统一要求，不因模型不同改变输出结构
- 错误处理统一：超时 → 重试 1 次 → 仍失败提示用户。速率限制统一走 Vercel KV
- API Key 全部通过 Vercel 环境变量管理，不暴露到前端
- 日志记录每次调用的 `provider` + `model` + `duration` + `tokens`，方便后续对比模型质量

**模型切换操作（改一行环境变量即可）：**
```
# 切到 DeepSeek
AI_PROVIDER=deepseek
AI_BASE_URL=https://api.deepseek.com/v1
AI_MODEL=deepseek-v4

# 切到 Qwen
AI_PROVIDER=qwen
AI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
AI_MODEL=qwen3.5-plus

# 切到 Gemini
AI_PROVIDER=gemini
AI_BASE_URL=https://generativelanguage.googleapis.com/v1beta/openai
AI_MODEL=gemini-2.5-flash

# 切到 Claude
AI_PROVIDER=claude
AI_BASE_URL=https://api.anthropic.com/v1
AI_MODEL=claude-sonnet-4-6
```

**输出层：**
- 用例表格展示（编号/标题/前置条件/步骤/预期结果/优先级/类型）
- 点击某条用例 → 展开内联编辑面板（可直接修改所有字段）
- 附加输出：规则模糊点标记（AI 标注需求中自相矛盾或未定义的地方）
- 导出：复制为纯文本（编号+标题+步骤的层级格式） / 下载 Excel（xlsx）

**UI 状态覆盖：**
- 加载态：生成中显示骨架屏 + 文字提示（"正在分析需求并生成用例…预计 15-30 秒"）
- 空态（首次使用）：展示一组示例输入和对应输出截图，引导用户"粘贴你的第一条需求"
- 错误态：API 失败显示具体错误信息 + "重试"按钮。导出失败显示"导出失败，请重试"
- 成功态：用例表格 + 内联编辑 + 导出按钮
- 响应式：桌面端优先（QA 主要工作场景），移动端保证基础可用（查看结果、复制文本）

**数据层：**
- localStorage 存使用次数、最近 10 条生成结果（仅存文本，不存图片 base64）
- 无服务端数据库（MVP 阶段）
- Vercel Analytics 做基础页面访问统计（PV/UV），不做自定义事件埋点

**不做（留到后续）：**
- 登录注册
- 数据库存储
- 团队协作/共享模板
- Jira/飞书/禅道集成
- 浏览器插件
- 付费系统
- 自定义 Prompt 偏好设置（MVP 用一个调优好的固定 Prompt）
- 拖拽上传图片（MVP 只做粘贴，拖拽后续加）

### JSON 输出 Schema

AI 必须返回以下结构，解析层做容错（去掉 markdown 代码块标记、修正尾部逗号等，重试 1 次仍失败则提示用户）：

```json
{
  "title": "需求标题",
  "summary": "一句话需求概括",
  "testCases": [
    {
      "id": "TC-001",
      "title": "用例标题",
      "precondition": "前置条件",
      "steps": ["步骤 1", "步骤 2"],
      "expected": "预期结果",
      "priority": "P0 | P1 | P2 | P3",
      "type": "功能 | 边界值 | 异常 | 兼容性 | 性能"
    }
  ],
  "fuzzyPoints": [
    {
      "description": "规则模糊或自相矛盾的地方",
      "suggestion": "建议向产品确认的问题"
    }
  ]
}
```

### AI 模型成本对比（月 500 次调用估算）

统一假设：每次调用 ~8K input + 2K output（文本 + 1 张图），月 500 次。

| 模型 | 单次成本 | 月成本（¥） | 能否国内直连 | 备注 |
|------|---------|-----------|------------|------|
| **Gemini 2.5 Flash-Lite** | ~$0.002 | ~¥6 | 需代理 | 每天 1000 次免费，可能完全不要钱 |
| **Gemini 2.5 Flash** | ~$0.007 | ~¥25 | 需代理 | 每天 250 次免费，够覆盖大部分 |
| **Qwen3.5-Plus** | ~¥0.016 | ~¥40 | ✅ 阿里云直连 | 新用户送 7000 万 token |
| **DeepSeek V4** | ~¥0.018 | ~¥45 | ✅ 直连 | JSON 输出最稳，OpenAI SDK 兼容 |
| **GPT-4.1 Mini** | ~$0.006 | ~¥45 | 需代理 | OpenAI 最便宜的视觉模型 |
| **Claude Haiku 4.5** | ~$0.014 | ~¥100 | 需代理 | Claude 生态入门 |
| **GPT-4o** | ~$0.020 | ~¥130 | 需代理 | JSON strict mode |
| **Claude Sonnet 4.6** | ~$0.042 | ~¥300 | 需代理 | 质量天花板 |

**MVP 推荐启动配置：** 先用 DeepSeek V4（国内直连、便宜、JSON 稳），后续按需切到 Qwen3.5-Plus（中文更自然）或 Claude Sonnet 4.6（追求质量时）。换模型 = 改一行环境变量，无需改代码。

### System Prompt 结构（初版大纲）

这是产品最核心的资产。Prompt 存为独立文件（`prompts/test-case-generator.md`），不硬编码。后续可按模型分版本（`prompts/test-case-generator-deepseek.md`、`prompts/test-case-generator-claude.md`），切换模型时自动加载对应 Prompt。

不在文档里写完整 Prompt（需要实测迭代），但结构如下：

```
# 角色
你是资深 QA 测试专家，有 10 年电商/金融/SaaS 行业测试经验。

# 任务
根据用户输入的需求描述，生成完整的结构化测试用例。

# 输出规则
1. 每个用例必须有：编号、标题、前置条件、步骤（数组）、预期结果、优先级、类型
2. 必须覆盖以下类型各至少 1 条：正向功能、边界值、异常场景
3. 优先级的判定标准：P0=核心流程阻断、P1=主要功能但可绕过、P2=边缘场景、P3=建议优化
4. 步骤用数组，每步一个字符串，按操作顺序排列
5. 如果需求中存在模糊或自相矛盾的地方，在 fuzzyPoints 字段中标注

# 特别关注
- 金额/数量字段：关注 0、负数、最大值、小数位数
- 文本输入：关注空字符串、超长、特殊字符、SQL/XSS 注入
- 并发/状态：关注重复提交、状态流转、超时
- 权限：关注无权限用户、跨租户隔离
```

### 测试策略（测试工具本身的质量）

作为 QA，你自然关心输出质量怎么保证。MVP 阶段：

1. **准备 5 份标准需求**（覆盖电商、SaaS、金融三个领域），每份有已知的高质量用例作为参考
2. 每次 Prompt 改动后，跑这 5 份需求，对比输出质量
3. 核心指标：AI 用例的"可直接采纳率"和"编辑后采纳率"
4. 目标：编辑后采纳率 ≥ 80%，可直接采纳率 ≥ 40%
5. 支持切换模型后重跑同一份需求，对比两个模型的输出质量（A/B 测试）

### 技术栈

```
前端：Next.js 14 (App Router) + Tailwind CSS + shadcn/ui 组件
后端：Next.js API Routes（同仓库，不需要单独后端服务）
AI：可插拔 Provider 抽象层（默认兼容 OpenAI SDK 格式，改环境变量即切换模型）
导出：exceljs 库（支持样式、合并单元格、中文不乱码）
部署：Vercel Pro（前端+服务端一体，需要 30s 函数超时）
监控：Vercel Analytics（PV/UV）
```

### 为什么选这个技术栈

- **一个人能驾驭：** Next.js 全栈在一个仓库里，不用分开部署前后端
- **部署低成本：** Vercel Pro $20/月搞定前端+后端+30s 超时，够 100-500 DAU
- **AI 辅助友好：** React + Tailwind + shadcn/ui 是 Claude Code 写的最高质量代码组合
- **后续从 MVP 到正式产品不停机：** 加数据库（Vercel Postgres）、加 Auth（NextAuth）、加 Stripe 都在同一框架内
- **AI 不锁定厂商：** 不用任何厂商专用 SDK，全部走标准 OpenAI SDK 兼容接口。DeepSeek、Qwen、Gemini、Claude、GPT 全部支持。换模型 = 改一行环境变量，不停机、不改代码、不用重新部署

## Open Questions

1. Claude API 单次调用成本实测——视觉模型处理多张图会比纯文本贵多少？
2. **国内用户访问 Vercel 部署的速度如何？** 这是最大的部署风险——如果速度不可接受，备选方案是阿里云函数计算 + 阿里云域名，或 Railway 等替代平台
3. Prompt 在不同类型需求（电商/金融/SaaS/游戏）上的输出质量差异多大？
4. 图片+文本组合输入 vs 纯文本输入的用例质量提升有多少？值得增加图片上传的 API 成本吗？

## Key Risks & Mitigations

| 风险 | 严重度 | 缓解措施 |
|------|--------|----------|
| AI 生成用例质量不达标（前提 2 未验证） | **高** | 第一周专注 Prompt 调优，同时用同一份需求测试 2-3 个模型对比输出质量。如果最佳模型采纳率仍 <60%，暂停开发 |
| 不同模型的 Prompt 兼容性 | **中** | 同一个 Prompt 在不同模型上效果可能不同。System Prompt 存为独立文件，按模型分版本（如 `test-case-generator-deepseek.md`），切换模型时同步切换 Prompt 文件 |
| Vercel 国内访问慢 | **中** | 部署后第一时间让国内用户测速。如果不可接受，迁移到阿里云函数计算或 Railway |
| AI JSON 解析失败率高 | **中** | 做两层容错：正则去掉 markdown 代码块 + JSON 语法修正。重试 1 次，失败则展示原始文本（降级体验）。切换模型时优先选 JSON 输出稳定的（DeepSeek、GPT-4o strict mode）|
| localStorage 丢失历史 | **低** | 不影响核心功能。后续加数据库前，提示用户"重要用例请导出保存" |

## Success Criteria

由于 MVP 无登录系统，指标通过以下方式手动跟踪：
- 第 1 周：自己每天用，至少生成 10 份真实需求的用例
- 第 2 周：发给 5-10 个 QA 同行试用，微信群/私聊收集反馈
- 第 4 周：有至少 3 个非朋友的用户每周主动使用（通过聊天询问 + Vercel Analytics UV 交叉验证）
- 核心指标：生成→用户手动编辑后的采纳率（发给用户后询问"50 条里你改了几条？"）

## What I noticed about how you think

- 你在选择方向时，问的是"哪个市场需求大"和"怎么变现"——说明你虽然在说 side project，内心想的是"这个东西能不能真的跑起来、有人付钱"。这个心态比"先做着玩玩"更容易做出有用的东西。
- 在 Bug 截图转报告和测试用例生成之间，你没纠结太久——因为你清楚自己的核心优势是 QA 经验，不是通用编程能力。这个自知之明很重要。
- 你会追问具体的实现细节（要不要登录、怎么做成网页），而不是停留在"想法很酷"的层面。这是 builder 的信号。
